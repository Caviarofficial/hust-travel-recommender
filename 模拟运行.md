# 改进算法模拟运行报告

## 0. 问题背景

我们的推荐系统分两个阶段：
1. **数据收集阶段**：让50名同学填写5道动态选择题，每人产生一个 path_code（如 `A-B-A-B-A`）
2. **推荐阶段**：一个全新用户只回答2道判断题（J1=距离、J2=体力），系统推荐一个景点

**核心难点**：填问卷的人和用推荐的人是不同的人。我们没有"某人回答J1/J2后去了哪个景点"的数据。
所以不能直接统计 P(景点 | J1, J2)。

**解决思路**：利用决策树的结构特性——path_code 的前两位天然编码了 J1 和 J2 的信息：
- 第1位（Q1）：A=近、B=远 → 对应 J1（距离）
- 第2位（Q2）：A=想放松、B=精力充沛 → 对应 J2（体力）

**关键改动**：原设计中 Q1=B（远）后的 Q2 问的是"核心目的（文化风景 vs 吃喝玩乐）"，
这与 J2（体力）的含义不匹配。改进后，无论近远，Q2 统一问"精力状态"，
使得 path_code 前两位与 J1、J2 严格对应。

| J1(远?) | J2(体力?) | 匹配的路径前缀 | 含义 |
|---------|-----------|---------------|------|
| 否 | 否 | A-A-\*-\*-\* | 近+放松 |
| 否 | 是 | A-B-\*-\*-\* | 近+活跃 |
| 是 | 否 | B-A-\*-\*-\* | 远+放松 |
| 是 | 是 | B-B-\*-\*-\* | 远+活跃 |

### 改动后的远距离问卷分支

**Q2（Q1=B 时）**：今天的精力状态如何？
- A. 想放松（散步、看展、拍照）
- B. 精力充沛（爬山、逛街、找美食、游乐场）

**Q3（B-A 远+放松）**：更想看什么？
- A. 自然风景（江滩、花园、公园）→ Q5区分时间
- B. 文化艺术（老街、校园、博物馆）→ Q5区分时间

**Q3（B-B 远+活跃）**：今天的重点是？
- A. 美食探店 → Q5区分预算
- B. 逛街/观光/游乐 → Q5区分预算

**Q4（B-A-A）**：想去哪个方向？ A.江边 B.湖边/山边 → Q5: A半天/B一整天
**Q4（B-A-B）**：更偏向？ A.历史建筑 B.艺术展览 → Q5: A半天/B一整天
**Q4（B-B-A）**：想吃什么风格？ A.街头小吃 B.商圈餐厅 → Q5: A免费低/B可以花
**Q4（B-B-B）**：更想去？ A.逛街购物 B.观光/游乐 → Q5: A免费低/B可以花

## Step 1：统计路径先验概率 P(path)

**含义**：从50份问卷中，统计每条路径被走过多少次。这反映了华科学生群体的真实偏好分布。

**为什么需要**：这是唯一来自真实数据的概率。路径被走得越多，说明这类偏好越普遍，推荐权重越高。

**拉普拉斯平滑**：给每条路径加1次虚拟计数，避免概率为0：

$$P(\text{path}) = \frac{\text{该path出现次数} + 1}{\text{总问卷数} + 32}$$

**含义**：用户回答J1和J2后，锁定8条匹配路径，将先验概率归一化为条件概率。

**为什么这样做**：J1和J2把32条路径筛到8条，8条之间的权重比完全由问卷数据决定。

$$P(\text{path} \mid J_1, J_2) = \frac{P(\text{path})}{\sum_{\text{匹配的8条path}} P(\text{path})}$$

## Step 3：计算每个景点的推荐概率 P(item | J1, J2)

**含义**：把路径级别的概率"传递"到景点级别。路径概率越高，其中的景点被推荐的概率越高。

**优先级加权**：同一路径中排在前面的景点权重更高（用 1/排名 衰减）：

$$P(\text{item}_i \mid \text{path}) = \frac{w_i}{\sum_j w_j}, \quad w_j = \frac{1}{\text{rank}_j}$$

**最终公式**：

$$P(\text{item}_i \mid J_1, J_2) = \sum_{\text{path}} P(\text{path} \mid J_1, J_2) \times P(\text{item}_i \mid \text{path})$$

## 算法优势总结

1. **数据来源合法**：只使用问卷中真实的 path_code 频率，不虚构任何标签
2. **Q2统一为能量维度**：近和远分支的Q2都问精力状态，与J2严格对应
3. **问卷数据真正起作用**：50份问卷决定了同一象限内8条路径的权重分配
4. **优先级保留**：路径内的项目排序通过 1/rank 加权保留
5. **可解释性强**：每一步计算都有明确的概率含义
